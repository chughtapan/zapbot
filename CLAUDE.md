# Zapbot

Plan-to-code workflow for teams. Globally installed at `~/.claude/skills/zapbot/`.
Per-project config generated by `zapbot-team-init`.

## Architecture

See [ARCHITECTURE.md](ARCHITECTURE.md) for diagrams and design decisions.

- **webhook-bridge** (bin/webhook-bridge.ts) — Bun HTTP server on port 3000
- **agent-orchestrator** — port 3001, manages worktrees and Claude Code agents
- **bin/zapbot-publish.sh** — creates/updates GitHub issues with plans
- **bin/share-link.ts** — generates plannotator share URLs
- **bin/zapbot-team-init** — onboards repos with config + labels + .env
- **bin/zapbot-update-check** — VERSION check against remote

## Commands

```bash
./setup                                              # Install tools
./bin/zapbot-team-init                               # Onboard a repo (run from project dir)
./start.sh [project-dir] [--no-ngrok]                # Start everything
./bin/zapbot-publish.sh <plan-file> --key <name>     # Publish a plan
./test/e2e-smoke.sh                                  # Run tests
```

## Skill routing

When the user's request matches an available skill, ALWAYS invoke it using the Skill
tool as your FIRST action.

Key routing rules:
- "publish plan", "share plan", "sync plan" → invoke zapbot-publish
